# TFG_Joaquin_Montero_Garcia
This repository contains the scripts developed by Joaquín Montero García during his Bachelor’s Thesis, titled ‘Design and development of a deep learning–based system for the segmentation of anatomical structures in endoscopic images'.

There are four different scripts: "preprocesado_datasets", "ensemble", "entrenamiento_modelos" and "dibujar_contornos". The first one contains several functions that have been designed in order to facilitate dataset's preproccesing tasks like organising frames and masks by their name, countering the different classes contained in a frame or in a video, or overlaying masks and frames, among other tasks. In every function it is included a line of commented code that explains to the viewer how to use the functions and what is the main objective. 

"dibujar_contornos" is a simple script created to draw the contour of the classes using the masks. You only need to provide the path to the frame and its corresponding mask, and it includes customizable parameters such as contour thickness or the color dictionary.

"entrenamiento_modelos" is one of the most important scripts of the project. This script has been created in order to design, train and evaluate using metrics different kind of deep learning models. This script implements a binary semantic segmentation training pipeline using PyTorch Lightning and Segmentation Models PyTorch. It trains one model per anatomical class by converting the grayscale mask into a binary mask for the target class. The pipeline includes deterministic seeding, Albumentations-based preprocessing, a custom dataset, Dice loss, metric tracking (IoU, sensitivity, specificity, precision, F1), and automatic saving of metrics, loss curves, and model weights. Dataset splits are defined via an Excel file, and each class is trained independently through the entrenar_clase() function. It also includes commented lines explaining and indicating changes that must be done in order to get a succesful training, like parts where is essential to provide an specific path or other tips, so it is recommended to pay attention to them. If the pipeline is not modified, the only thing to adjust to train models is the function entrenar_clase() call, where we need to provide the target class to segment, the architecture to be used, the encoder to implement in case we would like to use one, and the weights of the encoder if are needed. 

The script named "ensemble" performs ensemble inference for binary semantic segmentation of a target anatomical class (by default, the anastomotic line) in colonoscopy images. It loads multiple pre-trained segmentation models built with segmentation_models_pytorch — each defined by its architecture and encoder — and runs them jointly over the validation and test splits defined in an Excel file. For each image, the models produce sigmoid probability maps that are averaged via soft voting, and a final binary prediction is obtained by applying a configurable decision threshold. A majority voting alternative is also implemented and can be swapped in if desired. Per-image metrics (IoU, F1, precision, TP/TN/FP/FN) are computed against the ground-truth masks, alongside global micro and macro aggregates. All results are exported to JSON files for downstream analysis. Reproducibility is ensured via fixed random seeds across Python, NumPy, and PyTorch. In order to use it, provide the combination of architectures, encoders and weights, apart from other aspects like the voting method, that can be changed. This script also includes commented lines guiding the viewer, so it is recommended to pay attention to them.
